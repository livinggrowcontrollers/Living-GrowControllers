# Buildozer / Kivy Android Build – Referenzdokumentation 2026

## Ziel

Diese Dokumentation basiert **ausschließlich auf praktisch durchgeführten Tests**. Es wurden keine Aussagen aus Foren, Blogs oder KI-Empfehlungen übernommen, sofern sie nicht durch eigene Builds verifiziert wurden.

---

# Ausgangssituation

Über mehrere Jahre wurde eine eingefrorene Buildumgebung verwendet.

Typische Merkmale:

* feste python-for-android-Version
* fest gepinnte Python-Version
* fest gepinnte Pillow-Version
* weitere fest gepinnte Abhängigkeiten
* Sorge vor neueren Python-Versionen

Dadurch entstand die Annahme, dass aktuelle Versionen von Buildozer bzw. python-for-android nicht mehr zuverlässig funktionieren.

Diese Annahme wurde im Rahmen dieser Tests überprüft.

---

# Neue Buildozer-Konfiguration

Die wichtigste Änderung bestand darin, sämtliche unnötigen Versionsbindungen zu entfernen.

Anstelle von

```ini
requirements = python3==3.11.9,setuptools,kivy,pyjnius,pillow==9.5.0,certifi,...
```

wurde verwendet

```ini
requirements = python3,setuptools,kivy,pyjnius,pillow,certifi,six,kivy_garden.graph,zeroconf,ifaddr
```

Außerdem wurde entfernt:

```ini
p4a.source_dir = ...
```

Dadurch verwendet Buildozer automatisch die aktuelle python-for-android-Version.

---

# Verwendete Android-Konfiguration

```ini
android.minapi = 29
android.ndk_api = 29
android.archs = arm64-v8a

android.sdk_path = ~/.buildozer/android/platform/android-sdk
android.ndk_path = ~/.buildozer/android/platform/android-ndk-r25b
```

SDK und NDK wurden bewusst beibehalten.

Der Test sollte ausschließlich die Buildumgebung überprüfen.

# Beobachtung während des Builds

Während des Buildprozesses erschienen mehrfach Meldungen wie

```
No matching distribution found for pyjnius==1.7.0
```

sowie

```
No matching distribution found for android
```

Diese Meldungen führten **nicht** zum Buildabbruch.

Die APK wurde dennoch vollständig erstellt.

Die Meldungen stammen aus einem internen pip-Dry-Run und waren in den durchgeführten Tests nicht kritisch.

---

# Härtetest

Durchgeführt wurde anschließend:

* buildozer clean
* Löschen des kompletten Projektordners `.buildozer`
* Neuer Download von python-for-android
* kompletter Neuaufbau

SDK und NDK blieben erhalten.

Auch dieser Test verlief erfolgreich.

# Erkenntnisse

✓ Python 3.14 kann als Ziel-Python erfolgreich verwendet werden.

✓ Host Python 3.11 funktioniert.
